## Section: FAQ

**How do you manage application updates across Android and iOS?**
Flutter's unified codebase synchronizes security hotfixes and core feature logic simultaneously. Store-review windows differ, so we plan for iOS's 24–48 hour review latency and Play's 1–24 hour review. Where a critical patch cannot wait, we use in-app config toggles to disable affected paths.

**Flutter or React Native?**
Default: Flutter. It delivers native-compiled performance, a single language (Dart), and predictable rendering. We recommend React Native when the client team is React-heavy and cross-platform performance is not the bottleneck. See our decision guide.
→ `/insights/flutter-vs-react-native/`

**Do you handle App Store and Play Store submissions?**
Yes — including account setup guidance, certificates, review responses, and ASO fundamentals.

**How do you protect user PII in the app?**
Encrypted local storage (Keychain/Keystore), certificate pinning on network requests, no PII in logs, and OWASP MASVS-aligned security review before launch.
