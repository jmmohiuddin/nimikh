## Section: FAQ

**Can headless architectures integrate with our legacy inventory systems?**
Yes. Our backend engineering layer maps custom API layers to pull tracking databases cleanly without site refactoring. We support ODBC, REST, SOAP, and CSV/SFTP feeds where legacy systems require them.

**Why not just use Shopify or WooCommerce?**
Both are excellent starting points. Move to headless when custom UX, sub-second load targets, or multi-surface delivery (web + app + kiosk) exceeds what template systems can deliver. See our decision guide.
→ `/insights/headless-commerce-vs-shopify/`

**How do you protect Core Web Vitals under real product-catalog load?**
Server-side render above the fold, ISR-cache catalog pages, use `next/image` with AVIF, preload the LCP image, defer analytics and chat via server-side GTM, and monitor with Chromium's `web-vitals` library on every route.

**Do we own the code?**
Yes, in every engagement. Source code is delivered in your repository from Sprint 0.

**What if we need to scale the team mid-project?**
ScaleGate lets you add or remove capacity per sprint, with 48-hour ramp on additional engineers from our talent pool.
